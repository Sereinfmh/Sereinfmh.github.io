Android开发：记事本

发表于2025-02-12|更新于2025-02-15
|浏览量:

使用Android Studio开发一个记事本，有注册、登录、添加、修改、删除、查询等核心功能模块。主要运用Kotlin、XML进行Android界面开发，采用SQLite数据库存储本地数据存储。

注册

用户在注册页面输入用户名、密码和确认密码，点击注册按钮后，程序会对输入信息进行验证，若验证通过则将用户信息保存到 SharedPreferences 中，并跳转到登录页面。

  1. Android 布局技术
    LinearLayout 布局管理：在 register.xml 中，主要使用了 LinearLayout 进行布局。最外层的 LinearLayout 的 android:orientation=”vertical”，这意味着其子视图会按照垂直方向依次排列，为整个注册页面提供了基础的布局框架。内部嵌套的 LinearLayout 用于将标题 “注册” 文本居中显示，通过设置 android:gravity=”center” 实现。
    视图控件的使用
    TextView：用于显示静态文本，在注册页面中用于显示 “注册” 标题，可通过设置 android:textSize、android:textStyle 和 android:textColor 等属性来调整文本的样式。
    EditText：提供了用户输入功能。分别创建了用于输入用户名、密码和确认密码的输入框，通过 android:id 为每个输入框设置唯一标识符，方便在 Java 代码中引用。android:inputType 属性指定了输入类型，如 text 用于用户名输入，textPassword 用于密码输入，保证密码输入时显示为密文。android:hint 属性为输入框提供提示信息，引导用户输入。
    Button：创建了 “注册” 按钮，通过 android:text 属性设置按钮上显示的文本，android:id 用于在 Java 代码中监听按钮的点击事件。
  2. Android 事件处理技术
    OnClickListener 监听器：在 RegisterActivity 的 onCreate 方法中，为注册按钮 btnRegister 设置了 OnClickListener 监听器。当用户点击注册按钮时，会触发 onClick 方法，进而调用 register 方法执行注册逻辑。
  3. Android 数据存储技术
    SharedPreferences：在 register 方法中，使用 SharedPreferences 来存储用户的注册信息（用户名和密码）。SharedPreferences 是 Android 提供的一种轻量级的数据存储方式，适合存储简单的键值对数据。
    getSharedPreferences(“login_info”, MODE_PRIVATE) 用于获取名为 login_info 的 SharedPreferences 对象，MODE_PRIVATE 表示该数据只能被当前应用访问。
    SharedPreferences.Editor 用于对 SharedPreferences 中的数据进行编辑操作，通过 putString 方法将用户名和密码存储到 SharedPreferences 中。
    editor.apply() 用于异步提交编辑操作，将数据保存到磁盘。
  4. Android 提示信息技术
    Toast 提示：在注册过程中，使用 Toast 类来显示提示信息给用户。当用户名、密码或确认密码为空时，显示 “用户名、密码和确认密码都不能为空”；当密码和确认密码不一致时，显示 “密码和确认密码不一致”；当注册成功时，显示 “注册成功”。
    makeText 方法用于创建一个 Toast 对象，第一个参数是上下文，第二个参数是要显示的提示信息，第三个参数是提示信息显示的时长（Toast.LENGTH_SHORT 表示短时长），最后调用 show 方法将提示信息显示出来。
  5. Android 界面跳转技术
    Intent 跳转：当用户注册成功后，使用 Intent 实现从 RegisterActivity 跳转到 LoginActivity。Intent 是 Android 中用于在不同组件（如活动、服务等）之间进行通信和跳转的重要机制。
    创建一个 Intent 对象，第一个参数是当前活动的上下文，第二个参数是要跳转的目标活动类。然后调用 startActivity 方法启动目标活动。
  6. 输入验证技术
    在 register 方法中，对用户输入的信息进行了验证。首先检查用户名、密码和确认密码是否为空，如果为空则通过 Toast 提示用户并返回，不进行后续操作。接着检查密码和确认密码是否一致，如果不一致同样通过 Toast 提示用户并返回。只有当所有验证都通过时，才会执行注册成功的操作。

登录

  1. Android 布局技术
    LinearLayout 布局：在 login.xml 中使用了 LinearLayout 进行界面布局。通过设置 android:orientation=”vertical”，实现了界面元素的垂直排列。这样可以方便地将登录标题、用户名输入框、密码输入框和登录按钮依次排列在界面上。
    嵌套的 LinearLayout 用于将登录标题文本居中显示，通过设置 android:gravity=”center” 实现。
    EditText 控件：使用 EditText 控件创建用户名和密码输入框。通过 android:id 为输入框设置唯一标识符，方便在 Java 代码中引用。例如 android:id=”@+id/etUsername” 和 android:id=”@+id/etPassword”。
    android:inputType 属性用于指定输入框的输入类型，如 text 用于用户名输入，textPassword 用于密码输入，保证密码输入时显示为密文。
    android:hint属性为输入框提供提示信息，帮助用户了解需要输入的内容。
    Button 控件：使用 Button 控件创建登录按钮，通过 android:text 属性设置按钮上显示的文本为 “登录”。同样通过 android:id 为按钮设置唯一标识符，以便在 Java 代码中监听按钮的点击事件。
  2. Android 事件处理技术
    OnClickListener 监听器：在 LoginActivity 的 onCreate 方法中，为登录按钮 btnLogin 设置了 OnClickListener 监听器。当用户点击登录按钮时，会触发 onClick 方法，进而调用 login 方法执行登录逻辑。
  3. Android 数据存储与读取技术
    SharedPreferences：在 login 方法中，使用 SharedPreferences 来读取之前注册时保存的用户名和密码信息。SharedPreferences 是 Android 提供的一种轻量级的数据存储方式，适合存储简单的键值对数据，如用户的登录信息。
  4. Android 提示信息技术
    Toast 提示：在登录过程中，使用 Toast 类来显示提示信息给用户。例如，当用户名或密码为空时，显示 “用户名和密码不能为空”；当用户名或密码错误时，显示 “用户名或密码错误”；当登录成功时，显示 “登录成功”。
  5. Android 界面跳转技术
    Intent 跳转：当用户登录成功后，使用 Intent 实现从 LoginActivity 跳转到 MainActivity。Intent 是 Android 中用于在不同组件（如活动、服务等）之间进行通信和跳转的重要机制。创建一个 Intent 对象，第一个参数是当前活动的上下文，第二个参数是要跳转的目标活动类。然后调用 startActivity 方法启动目标活动。

主页面

  1. Android 布局与视图绑定
    布局文件加载：
    在 onCreate 方法中，通过 setContentView(R.layout.activity_notepad) 加载了名为 activity_notepad 的布局文件，以此来定义主页面的 UI 结构。
    视图控件查找：
    使用 findViewById 方法来查找布局文件中的视图控件。例如，listView = (ListView) findViewById(R.id.listview) 找到了 ListView 控件，用于展示便签列表；ImageView add = (ImageView) findViewById(R.id.add) 找到了添加便签的图标按钮。
  2. 事件处理机制
    点击事件处理：
    为添加便签的图标按钮设置了点击事件监听器。当用户点击该按钮时，会创建一个 Intent 对象，用于启动 EditNoteActivity 活动，并且使用 startActivityForResult 方法启动该活动，以便在返回结果时能进行相应处理。
    列表项点击事件处理：
    为 ListView 的列表项设置了点击事件监听器。当用户点击某一便签项时，会获取该便签的信息（如 id、time、content），并将这些信息通过 Intent 传递给 EditNoteActivity 活动，同时启动该活动。
    列表项长按事件处理：
    为 ListView 的列表项设置了长按事件监听器。当用户长按某一便签项时，会弹出一个 AlertDialog 对话框，询问用户是否删除该便签。如果用户点击 “确定”，则调用数据库的 deleteData 方法删除该便签数据，同时从列表中移除该便签项，并通知适配器数据发生了变化，最后显示删除成功的提示信息；如果用户点击 “取消”，则关闭对话框。
  3. 数据库操作技术
    数据库创建与使用：
    在 initData 方法中，创建了 NoteDatabase 数据库的实例，用于进行便签数据的存储和管理。
    数据查询：
    showQueryData 方法从数据库中查询所有便签数据，并将查询结果存储在 list 列表中。然后创建一个 NotepadAdapter 适配器，将数据与 ListView 进行绑定，从而在界面上展示便签列表。
    数据删除：
    当用户在 ListView 中长按某一便签项并确认删除时，调用 database.deleteData(note.getId()) 方法从数据库中删除该便签数据。
  4. 活动结果处理技术
    onActivityResult 方法：
    重写了 onActivityResult 方法，用于处理从 EditNoteActivity 返回的结果。当 requestCode 为 1 且 resultCode 为 2 时，说明在 EditNoteActivity 中进行了数据的添加或修改操作，此时调用 showQueryData 方法重新查询数据库并更新 ListView 中的数据。
  5. 数据适配技术
    适配器使用：
    使用 NotepadAdapter 适配器将 Note 数据列表与 ListView 进行绑定。适配器负责将数据转换为 ListView 可以显示的视图，从而实现便签数据的展示。
  6. 提示信息显示技术
    Toast 提示：
    在删除便签成功后，使用 Toast.makeText(MainActivity.this, “删除成功”, Toast.LENGTH_SHORT).show() 方法显示一个短暂的提示信息，告知用户操作结果。
  7. 对话框显示技术
    AlertDialog 使用：
    在用户长按便签项时，使用 AlertDialog 对话框询问用户是否删除该便签。通过 AlertDialog.Builder 构建对话框，设置提示信息、“确定” 和 “取消” 按钮及其点击事件，最后创建并显示对话框。
